# Лабораторная работа: Автотесты с Cypress (Saucedemo)

Автотесты на Cypress для сайта [Saucedemo](https://www.saucedemo.com/).  
Выполнены по заданиям: проверка товаров, корзины и сортировки для стандартного и проблемного пользователей.

## Структура проекта

- `cypress/e2e/` – spec-файлы с тестами (по одному на задание).
- `cypress.config.js` – конфигурация Cypress.
- `package.json` – зависимости и скрипты.

## Как запустить

1. Открой терминал в этой папке (где лежит этот README).
2. Убедись, что зависимости установлены:
   ```bash
   npm install
   ```
3. Запусти Cypress в интерактивном режиме:
   ```bash
   npx cypress open
   ```
4. В окне Cypress выбери **E2E Testing**.
5. Выбери браузер **Chrome** и нажми **Start E2E Testing in Chrome**.
6. В открывшемся списке spec-файлов по очереди запускай нужные тесты.

## Список тестов и ожидаемое поведение

### 📦 Тест 1 (task1_standard_user.cy.js) — стандартный пользователь, проверка товара

**Сценарий:**
1. Зайти на сайт, ввести логин/пароль `standard_user / secret_sauce`, нажать Login.
2. Убедиться, что нас перебросило на страницу товаров (URL содержит `/inventory`).
3. Найти первый товар в списке и проверить, что его название — `Sauce Labs Backpack`, а цена — `$29.99`.
4. Кликнуть по названию первого товара, чтобы открыть карточку.
5. Проверить, что URL изменился на `/inventory-item`.
6. Внутри карточки снова проверить название и цену — они должны совпадать.

**Ключевые детали:**
- Использовались `[data-test="..."]` атрибуты — это специальные метки, которые разработчики добавили в HTML специально для автотестов. Они не привязаны к цвету, тексту или месту кнопки, поэтому тесты не ломаются при смене дизайна.
- Для доступа к первому элементу использовали `.eq(0)` (отсчёт с нуля). Без этого Cypress нашёл бы все шесть товаров и не понял бы, какой именно проверять.
- `cy.wait(1000)` добавлены только для наглядности в демонстрации, чтобы ты мог видеть, как заполняются поля и переключаются страницы. В реальных проектах так делать не рекомендуется — ждать нужно конкретных элементов или сетевых запросов.

✅ **Ожидаемый результат:** тест прошёл.

### 🛒 Тест 3 (task3_cart_standard.cy.js) — добавление/удаление товара (стандартный)

**Сценарий:**
1. Логинимся стандартным пользователем.
2. Проверяем, что кнопка «Add to cart» для товара `Sauce Labs Bolt T-Shirt` видна на странице.
3. Нажимаем её.
4. Проверяем, что значок на иконке корзины показывает `1` (это подтверждает, что товар добавился).
5. Теперь нажимаем кнопку «Remove» для того же товара (она появляется на месте «Add to cart» после добавления).
6. Проверяем, что значок на корзине **исчез** (`should('not.exist')`), то есть корзина пуста.

**Ключевые детали:**
- Атрибут `data-test="shopping-cart-badge"` ищет значок с количеством товаров. Если его нет в DOM (корзина пуста), Cypress подтвердит это через `should('not.exist')`.
- Использована `.should('be.visible')` для проверки, что кнопка добавления действительно есть на странице до клика.

✅ **Ожидаемый результат:** тест прошёл.

### 🔀 Тест 5 (task5_sorting.cy.js) — проверка сортировки (стандартный)

**Сценарий:**
1. Логинимся стандартным пользователем.
2. Проверяем, что изначально первый товар — `Sauce Labs Backpack` ($29.99) — это стандартная сортировка «Name (A to Z)».
3. Меняем сортировку на «Name (Z to A)» (от Я до А).
4. Проверяем, что теперь **первый** товар — `Test.allTheThings() T-Shirt (Red)` ($15.99).
5. Проверяем, что **последний** (шестой, индекс 5) товар — `Sauce Labs Backpack` ($29.99).

**Ключевые детали:**
- Элемент сортировки — выпадающий список `<select>`. Cypress умеет выбирать опцию по тексту: `.select('Name (Z to A)')`.
- Использовали `.eq(5)` для последнего элемента, так как всего товаров шесть.
- Цены у этих товаров разные, что дополнительно подтверждает, что сортировка реально изменилась.

✅ **Ожидаемый результат:** тест прошёл.
